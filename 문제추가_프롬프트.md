# 📋 문제 추가 프롬프트 (복사해서 매번 사용)

새 대화를 열고 아래 프롬프트를 통째로 복사 → 붙여넣기 → PDF나 텍스트 자료를 함께 첨부해서 보내면,
Claude가 자동으로 내 서버가 원하는 형식(JSON)으로 문제를 뽑아줍니다.
그 결과를 그대로 curl 명령어로 실행하면 서버 DB에 바로 추가됩니다.

---

## ⬇️ 여기부터 복사 ⬇️

너는 내 "정보처리산업기사 실기 학습 서버"에 넣을 문제 데이터를 만드는 도우미야.
첨부하거나 붙여넣는 자료(PDF, 텍스트, 이미지)에서 학습 문제가 될 만한 내용을 최대한 많이 뽑아서,
아래 JSON 형식으로 정확히 변환해줘.

## 문제 유형 4가지
1. **code**: 코드 트레이싱 문제 (C/Java/Python 실행 결과 맞히기)
   - lang: "C" | "Java" | "Python"
   - prompt: 코드 원문 (줄바꿈 포함)
   - answer: 정확한 실행 결과 (공백/줄바꿈 최소화해서 비교하니 너무 예민하게 신경 안 써도 됨)
   - explain: 왜 그 결과가 나오는지 2~3문장

2. **sql**: SQL 작성 문제
   - lang: null
   - prompt: "~하는 쿼리/조건절을 작성하세요" 형태의 질문
   - answer: 예상 정답 쿼리 (표준적인 한 가지 형태로)
   - explain: 핵심 문법 포인트 1~2문장

3. **etc**: OS / 네트워크 / 테스트기법 등 그 외 서술형·계산형 문제 (SQL과 같은 질문+답 형식)
   - lang: null
   - prompt: "~를 구하세요" / "~은 무엇인가?" 형태의 질문 (스케줄링·페이지 교체 계산, 서브넷 계산, 프로토콜, 테스트 기법, 쉘 명령어 등)
   - answer: 정답. 숫자 계산 문제는 정답이 소수/분수로 애매해지지 않도록 "합계를 구하세요"처럼 정수로 딱 떨어지게 질문을 다듬어줘 (정답은 문자열 완전일치로 채점됨)
   - explain: 계산 과정이나 핵심 포인트 1~2문장

4. **term**: 용어 카드
   - lang: null
   - prompt: 용어 이름
   - answer: 그 용어의 정의 (1~2문장)
   - explain: "" (빈 문자열로 둬도 됨)

## 출력 규칙
- 반드시 아래 JSON 스키마 그대로, 다른 설명 없이 **코드블록 안에 JSON만** 출력해.
- 정보처리산업기사 실기 출제기준(OS 기초, DB/SQL, 네트워크 기초, 프로그래밍 언어 기초, 테스트/배포)에
  해당하는 내용 위주로 뽑아줘. 디자인패턴/UML/암호알고리즘 같은 산업기사 범위 밖 내용은 제외해줘.
- 문제 개수는 자료에 있는 만큼 최대한 많이. 부족하면 있는 만큼만.

```json
{
  "admin_key": "여기에_내_ADMIN_KEY_입력",
  "problems": [
    {"kind":"code", "lang":"Java", "prompt":"...", "answer":"...", "explain":"..."},
    {"kind":"sql", "prompt":"...", "answer":"...", "explain":"..."},
    {"kind":"etc", "prompt":"...", "answer":"...", "explain":"..."},
    {"kind":"term", "prompt":"...", "answer":"...", "explain":""}
  ]
}
```

마지막에 이 JSON을 서버에 바로 넣을 수 있는 curl 명령어도 함께 만들어줘:

```bash
curl -X POST https://내서버주소.onrender.com/api/admin/add_problems \
  -H "Content-Type: application/json" \
  -d '위에서 만든 JSON 그대로'
```

### 사용법 요약
1. 위 프롬프트 전체 복사
2. 새 Claude 대화 시작
3. 프롬프트 붙여넣기 + PDF/텍스트 자료 첨부해서 전송
4. Claude가 만들어준 curl 명령어를 터미널(또는 컴퓨터)에서 실행
5. 서버 문제 DB에 바로 추가 완료 (재배포 필요 없음!)

### 참고
- `ADMIN_KEY`는 Render 환경변수에 설정한 값과 동일해야 해요 (README 참고, 기본값은 `changeme`이니 꼭 바꿔서 쓰세요).
- curl이 낯설면, 터미널 없이도 Postman이나 온라인 REST 클라이언트로 같은 요청을 보낼 수 있어요.
- 같은 kind + 같은 prompt(완전히 동일한 문제)는 서버가 자동으로 중복 스킵해줘서 여러 번 실행해도 안전해요.
